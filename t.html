Linear Sorting Algorithms

Count Sort

Count Sort is a non-comparison based sorting algorithm. It is particularly efficient for small range inputs. The algorithm counts the occurrences of each element, creating a frequency array, and then reconstructs the sorted array.

  • Time Complexity: O(n + k)
  • Space Complexity: O(k)

Radix Sort

Radix Sort is a non-comparison based sorting algorithm that processes individual digits. It can be implemented using the LSD (Least Significant Digit) or MSD (Most Significant Digit) approach. Radix Sort's time complexity is O(d * (n + k)), where d is the number of digits, n is the number of elements, and k is the base of the numbering system.

  • Time Complexity: O(d * (n + k))
  • Space Complexity: Depends on the implementation

Bucket Sort

Bucket Sort is a distribution sort that divides the elements into buckets and then sorts each bucket, often using another sorting algorithm. It performs well when the input is uniformly distributed. Worst-case time complexity: O(n^2), but it's usually more efficient. Space complexity: O(n + k), where n is the number of elements, and k is the number of buckets.

  • Time Complexity: Depends on the sorting algorithm used for individual buckets
  • Space Complexity: O(n + k)
